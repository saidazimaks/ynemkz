Ты — инженер данных проекта Ynem. Твоя задача — наполнить БД правдоподобными демо-партнёрами, чтобы витрина, карта и скидка дня в Mini App выглядели живыми.

## Контекст

- БД — Supabase Postgres, доступ обычным `psql "$DATABASE_URL"` (переменная из `.env`). Инструмента миграций нет; сиды-скрипты клади в `migrations/` с префиксом `seed_` (например `migrations/seed_demo_partners.sql`), чтобы их можно было перезапустить.
- Схема таблицы `partners` (001_init.sql + 002_miniapp.sql): `user_id` (bigint, владелец — можно NULL для демо), `name`, `category`, `address`, `discount_free` (default 5), `discount_premium` (10–15), `avg_check`, `work_hours`, `is_active`, `is_paused`, `lat`, `lng`, `logo_url`.
- Скидка дня — строка в `daily_deals (partner_id, deal_date UNIQUE, description)`; назначь одного из демо-партнёров на сегодняшнюю дату (`CURRENT_DATE`), иначе главный экран пустой.
- Город — Экибастуз, центр примерно lat 51.723, lng 75.323. Координаты пинов разброси в радиусе ~2 км, чтобы карта смотрелась естественно.

## Правила для данных

- 2–3 партнёра разных категорий (например: кофейня, автомойка, салон красоты) с реалистичными казахстанскими названиями, адресами улиц Экибастуза, часами работы и средним чеком в тенге.
- `logo_url` — прямые ссылки на фото из интернета, стабильные и hotlink-доступные: `https://images.unsplash.com/photo-...?w=400&q=80` (Unsplash CDN отдаёт без ключей). Перед вставкой проверь каждую ссылку через `curl -sI <url>` — статус должен быть 200 и `content-type: image/*`.
- Сид должен быть идемпотентным: `INSERT ... ON CONFLICT DO NOTHING` либо проверка существования по `name`, чтобы повторный запуск не плодил дубли.
- Ничего не удаляй и не изменяй у существующих партнёров.

## Проверка

После вставки: `SELECT id, name, category, lat, lng, logo_url FROM partners` — убедись, что демо-строки на месте и координаты валидны (эндпоинты API требуют initData, поэтому проверка — на уровне SQL). В финальном сообщении приведи вставленные строки и путь к сид-файлу.
